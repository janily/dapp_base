# wagmi 功能实现总结

本文档总结 `wagmi` 示例应用中 5 个核心功能模块的实现逻辑，以及每个模块使用到的 wagmi Hooks 和 viem 工具方法。wagmi 本身是 React Hooks 封装，底层仍大量使用 viem 的类型、ABI 解析、单位转换与 public client 能力。

## 公共配置与基础封装

### 配置来源

- `appConfig` 从 `VITE_WAGMI_*` 或通用 `VITE_*` 环境变量读取：`CHAIN_ID`、`CHAIN_NAME`、`RPC_URL`、`READ_RPC_URL`、`ERC20_ADDRESS`、`DEFAULT_QUERY_ADDRESS`、`DEFAULT_RECIPIENT`。
- 默认链为 Sepolia：`chainId = 11155111`。
- ERC-20 ABI 使用 viem 风格的 typed ABI，供 `useReadContract` 和 `useWriteContract` 复用。

### wagmi 配置

`src/config.ts` 中通过以下方式创建 wagmi 配置：

1. 使用 viem 的 `defineChain(...)` 定义当前链。
2. 使用 `createConfig(...)` 创建 wagmi 全局配置。
3. `chains` 指定配置链。
4. `connectors` 配置注入式钱包连接器：优先 MetaMask，再添加通用 injected connector。
5. `transports` 使用 viem 的 `http(appConfig.readRpcUrl)` 指定链对应的 RPC transport。

### 关键 wagmi / viem 方法

| 方法/Hooks | 来源 | 作用 |
| --- | --- | --- |
| `createConfig` | wagmi | 创建 wagmi 配置 |
| `injected` | wagmi/connectors | 使用浏览器注入钱包连接器 |
| `useAccount` | wagmi | 获取当前钱包地址与连接状态 |
| `useConnect` / `useDisconnect` | wagmi | 连接/断开钱包 |
| `useChainId` / `useSwitchChain` | wagmi | 获取当前链、切换到目标链 |
| `useBalance` | wagmi | 查询原生币余额 |
| `useSendTransaction` | wagmi | 发送原生币交易 |
| `useWaitForTransactionReceipt` | wagmi | 等待交易确认 |
| `useReadContract` | wagmi | 读取合约函数 |
| `useWriteContract` | wagmi | 写入合约函数 |
| `usePublicClient` | wagmi | 获取底层 viem public client |
| `defineChain` / `http` | viem | 定义链与 RPC transport |
| `parseAbiItem` | viem | 解析事件 ABI，供日志查询使用 |
| `isAddress` | viem | 校验地址 |

| `formatUnits` | viem | Token 数量格式化 |

### 钱包账号切换补充逻辑

wagmi 的 `useConnect` 负责连接钱包，但示例中的“切换账号”按钮需要主动让 injected 钱包重新弹出账户选择，因此额外实现了 `requestInjectedAccountSelection()`：

1. 检查 `window.ethereum`。
2. 调用 `wallet_requestPermissions` 重新请求 `eth_accounts` 权限。
3. 调用 `eth_requestAccounts` 获取用户选择后的账号。
4. 返回第一个账号给页面状态使用。

## 1. 连接测试网并查询 ETH 余额

### 实现逻辑

1. `WalletAndEthBalance` 使用 `useAccount()` 获取当前钱包地址 `address` 和连接状态 `isConnected`。
2. 使用 `useConnect()` 获取所有 connectors，并在未连接时渲染连接按钮。
3. 已连接时使用 `useDisconnect()` 提供断开钱包能力。
4. 使用 `useChainId()` 读取当前链 ID。
5. 如果当前链不是配置链，用户可点击按钮调用 `useSwitchChain()` 返回的 `switchChain({ chainId })` 切换到目标链。
6. 查询地址默认使用环境变量中的 `defaultQueryAddress`，如果为空则在钱包连接后填入当前钱包地址。
7. 使用 `isAddress(queryAddress)` 判断查询地址是否合法，并作为 `useBalance` 的启用条件。
8. `useBalance({ address, query: { enabled } })` 负责查询 ETH 余额；点击“查询余额”时调用 `refetch()` 重新请求。
9. wagmi 返回的 `data.formatted` 和 `data.symbol` 直接用于页面展示。

### 使用到的 wagmi / viem 方法

- `useAccount()`：读取连接状态和当前钱包地址。
- `useConnect()`：获取连接器并连接钱包。
- `useDisconnect()`：断开钱包。
- `useChainId()`：读取当前链 ID。
- `useSwitchChain()`：切换到配置链。
- `useBalance()`：查询原生币余额。
- `refetch()`：手动刷新余额查询。
- `isAddress(value)`：校验查询地址。

## 2. 发送 ETH

### 实现逻辑

1. 用户输入收款地址和 ETH 数量。
2. 使用 `isAddress(to)` 和 `Number(amount) > 0` 计算 `canSend`，控制按钮启用状态。
3. `useSendTransaction()` 返回 `sendTransaction`、交易哈希 `hash`、错误和 pending 状态。
4. 点击“发送 ETH”时调用：
   `sendTransaction({ to, value: parseEther(amount) })`。
5. `parseEther(amount)` 将 ETH 字符串转换为 wei。
6. `useWaitForTransactionReceipt({ hash })` 根据交易哈希等待回执。
7. `receipt.isSuccess` 为 true 时展示“交易已确认”。

### 使用到的 wagmi / viem 方法

- `useSendTransaction()`：发送原生 ETH 交易。
- `sendTransaction({ to, value })`：触发钱包签名并广播交易。
- `useWaitForTransactionReceipt({ hash })`：监听交易确认状态。
- `parseEther(amount)`：ETH 数量转 wei。
- `isAddress(value)`：校验收款地址。

## 3. 调用 ERC-20 `balanceOf`

### 实现逻辑

1. ERC-20 合约地址来自环境变量或页面“配置合约地址”弹窗。
2. 用户输入要查询 Token 余额的账户地址。
3. `enabled = isAddress(erc20Address) && isAddress(account)` 控制查询是否启用。
4. 调用 `useReadContract(...)`：
   - `address` 为合法合约地址。
   - `abi` 为 `erc20Abi`。
   - `functionName` 为 `balanceOf`。
   - `args` 为 `[account]`。
   - `query.enabled` 控制是否自动/允许查询。
5. 点击“读取 Token 余额”时调用 `refetch()` 手动刷新。
6. 如果返回值是 bigint，使用 `formatUnits(data, 18)` 按 18 位小数格式化显示。

### 使用到的 wagmi / viem 方法

- `useReadContract(...)`：读取 ERC-20 view 函数。
- `refetch()`：手动刷新合约读取结果。
- `formatUnits(data, 18)`：格式化 Token 最小单位。
- `isAddress(value)`：校验合约地址和账户地址。

## 4. 监听 ERC-20 `Transfer` 事件

### 实现逻辑

1. 顶部使用 `parseAbiItem('event Transfer(...)')` 解析 `Transfer` 事件 ABI。
2. `TransferEvents` 组件用 `usePublicClient()` 获取 wagmi 底层的 viem public client。
3. `canWatch = Boolean(publicClient) && isAddress(erc20Address)` 控制是否启动监听。
4. `appendLogs` 用 `useCallback` 封装日志追加逻辑：
   - 读取每条日志的 `from`、`to`、`value` 参数。
   - 使用 `formatUnits(value, 18)` 展示 Token 数量。
   - 新日志放在列表顶部，最多保留最近 20 条。
5. effect 启动 `pollTransferLogs()`：
   - 调用 `client.getBlockNumber()` 获取最新区块。
   - 初次扫描默认回看最近 5 个区块，避免页面刚打开时完全错过附近事件。
   - 调用 `client.getLogs({ address, event, fromBlock, toBlock })` 按区块范围查询 Transfer 日志。
   - 更新 `lastCheckedBlock`，页面展示已检查到的区块。
   - 每 4 秒继续轮询。
6. effect 清理阶段设置 `cancelled = true` 并清理定时器，避免卸载后继续请求或更新状态。

该模块刻意使用 `eth_getLogs` 风格的区块范围轮询，而不是依赖 `eth_newFilter` / `eth_getFilterChanges`，因为公共 RPC 的 filter 可能过期或丢失。

### 使用到的 wagmi / viem 方法

- `usePublicClient()`：获取当前 wagmi 配置下的 viem public client。
- `parseAbiItem(...)`：解析 Transfer 事件 ABI。
- `client.getBlockNumber()`：获取最新区块号。
- `client.getLogs({ address, event, fromBlock, toBlock })`：按区块范围查询事件日志。
- `formatUnits(value, 18)`：格式化事件中的 Token 数量。
- `isAddress(value)`：监听前校验合约地址。

## 5. ERC-20 Token 转账

### 实现逻辑

1. 用户输入接收地址和 Token 数量。
2. `canTransfer = isAddress(erc20Address) && isAddress(to) && Number(amount) > 0` 控制按钮启用状态。
3. `useWriteContract()` 返回 `writeContract`、交易哈希 `hash`、错误和 pending 状态。
4. 点击“转账 Token”时调用：
   `writeContract({ address, abi: erc20Abi, functionName: 'transfer', args: [to, parseUnits(amount, 18)] })`。
5. `parseUnits(amount, 18)` 将 Token 数量转换为 ERC-20 最小单位。
6. `useWaitForTransactionReceipt({ hash })` 等待交易确认。
7. `receipt.isSuccess` 为 true 时展示“转账已确认”。

### 使用到的 wagmi / viem 方法

- `useWriteContract()`：调用 ERC-20 写入函数。
- `writeContract({ address, abi, functionName, args })`：触发钱包签名并广播合约交易。
- `useWaitForTransactionReceipt({ hash })`：等待交易确认。
- `parseUnits(amount, 18)`：Token 数量转最小单位。
- `isAddress(value)`：校验合约地址和接收地址。
