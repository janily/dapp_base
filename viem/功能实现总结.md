# viem 功能实现总结

本文档总结 `viem` 示例应用中 5 个核心功能模块的实现逻辑，以及每个模块使用到的 viem 方法。应用整体通过 `src/config.ts` 定义链信息、创建 `publicClient`，并在 `src/App.tsx` 中使用 `walletClient` 和 `publicClient` 分别处理写入和读取能力。

## 公共配置与基础封装

### 配置来源

- `appConfig` 从 `VITE_VIEM_*` 或通用 `VITE_*` 环境变量读取：`CHAIN_ID`、`CHAIN_NAME`、`RPC_URL`、`READ_RPC_URL`、`ERC20_ADDRESS`、`DEFAULT_QUERY_ADDRESS`、`DEFAULT_RECIPIENT`。
- 默认链为 Sepolia：`chainId = 11155111`。
- `readRpcUrl` 默认使用 `/rpc/sepolia`，`rpcUrl` 用于钱包添加链时写入钱包配置。

### 关键 viem 封装

| 封装/方法 | 作用 | 使用场景 |
| --- | --- | --- |
| `defineChain(...)` | 定义当前应用使用的链信息 | `publicClient`、`walletClient`、钱包加链 |
| `createPublicClient({ chain, transport })` | 创建只读客户端 | ETH 余额、ERC-20 读取、事件轮询、等待交易确认 |
| `http(appConfig.readRpcUrl)` | 创建 HTTP RPC transport | 只读链上请求 |
| `custom(window.ethereum)` | 将浏览器钱包 EIP-1193 Provider 转为 viem transport | 创建 `walletClient` |
| `createWalletClient({ chain, transport })` | 创建钱包客户端 | 请求账号、发送 ETH、写入合约 |
| `isAddress(value)` | 校验地址是否合法 | 表单校验、按钮禁用 |
| `parseEther` / `formatEther` | ETH 字符串与 wei bigint 互转 | ETH 转账、ETH 余额展示 |
| `parseUnits` / `formatUnits` | Token 字符串与最小单位互转 | ERC-20 转账、Token 余额/事件展示 |

### 钱包切链逻辑

`switchToConfiguredChain()` 直接使用浏览器钱包的 EIP-1193 RPC：

1. 将 `appConfig.chainId` 转成十六进制 `chainId`。
2. 调用 `wallet_switchEthereumChain` 切换链。
3. 如果返回 `4902`，说明钱包尚未添加该链，再调用 `wallet_addEthereumChain` 添加链配置。
4. 如果没有浏览器钱包，抛出错误提示用户安装 MetaMask 或兼容钱包。

### 钱包账户获取

`getWalletAccount(forceAccountPicker)` 是 viem 写操作的入口：

1. 先调用 `switchToConfiguredChain()`。
2. 如果需要切换账号，调用 `wallet_requestPermissions` 让钱包重新选择账户。
3. 通过 `createWalletClient({ chain: configuredChain, transport: getWalletTransport() })` 创建钱包客户端。
4. 调用 `walletClient.requestAddresses()` 请求/读取钱包授权地址。
5. 返回 `{ walletClient, account }`，供 ETH 转账和 ERC-20 写入复用。

## 1. 连接测试网并查询 ETH 余额

### 实现逻辑

1. 用户点击“连接钱包”或“切换账号”。
2. 调用 `getWalletAccount(forceAccountPicker)`：切链、可选账号选择、创建 `walletClient`、请求钱包地址。
3. 将返回的 `account` 写入当前钱包状态；如果查询地址为空，则把当前钱包地址作为默认查询地址。
4. 用户输入要查询的地址，按钮通过 `isAddress(queryAddress)` 控制是否可点击。
5. 点击“查询余额”后调用 `publicClient.getBalance({ address })`。
6. 使用 `formatEther(value)` 将 wei 格式余额转成 ETH 字符串展示。

### 使用到的 viem 方法

- `createWalletClient({ chain, transport })`：创建浏览器钱包客户端。
- `custom(window.ethereum)`：把钱包注入对象转成 viem transport。
- `walletClient.requestAddresses()`：请求/读取钱包账号。
- `publicClient.getBalance({ address })`：读取原生币余额。
- `formatEther(value)`：格式化 ETH 余额。
- `isAddress(value)`：校验查询地址。

## 2. 发送 ETH

### 实现逻辑

1. 用户输入接收地址和 ETH 数量。
2. 按钮启用条件：接收地址合法且数量大于 0。
3. 点击“发送 ETH”后，通过 `getWalletAccount()` 获取 `walletClient` 和 `account`。
4. 使用 `parseEther(amount)` 将 ETH 字符串转换成 wei。
5. 调用 `walletClient.sendTransaction({ account, to, value })` 发起原生币交易。
6. viem 返回交易哈希字符串 `txHash`，页面将其展示。
7. 调用 `publicClient.waitForTransactionReceipt({ hash: txHash })` 等待交易确认。

### 使用到的 viem 方法

- `walletClient.sendTransaction({ account, to, value })`：发送原生 ETH 交易。
- `parseEther(amount)`：ETH 数量转 wei。
- `publicClient.waitForTransactionReceipt({ hash })`：等待交易回执。
- `isAddress(value)`：校验收款地址。

## 3. 调用 ERC-20 `balanceOf`

### 实现逻辑

1. ERC-20 合约地址由环境变量或页面弹窗配置。
2. 用户输入需要查询 Token 余额的账户地址。
3. 按钮启用条件：合约地址和账户地址均合法。
4. 点击“读取 Token 余额”后，调用：
   `publicClient.readContract({ address, abi: erc20Abi, functionName: 'balanceOf', args: [account] })`。
5. `readContract` 根据 ABI、函数名和参数组装 `eth_call`，返回 `uint256` 对应的 bigint。
6. 使用 `formatUnits(value, 18)` 按 18 位小数格式化显示。

### 使用到的 viem 方法

- `publicClient.readContract(...)`：读取合约 view/pure 方法。
- `formatUnits(value, 18)`：格式化 Token 最小单位。
- `isAddress(value)`：校验合约地址和账户地址。

## 4. 监听 ERC-20 `Transfer` 事件

### 实现逻辑

1. `TransferEvents` 组件在 `erc20Address` 变化时重置事件列表和错误状态。
2. 合约地址无效时直接返回，不启动轮询。
3. 启动 `pollTransferEvents()` 轮询函数：
   - 调用 `publicClient.getBlockNumber()` 获取当前最新区块。
   - 第一次执行时只记录当前区块，避免一次性拉取大量历史日志。
   - 如果最新区块大于上次扫描区块，则调用 `publicClient.getContractEvents(...)` 查询新区块范围内的 `Transfer` 事件。
   - 将日志参数 `log.args.from`、`log.args.to`、`log.args.value` 格式化为页面文本。
   - 事件列表只保留最近 20 条。
4. 使用 `setTimeout(pollTransferEvents, publicClient.pollingInterval)` 按 viem 客户端的轮询间隔继续检查新区块。
5. React effect 清理阶段设置停止标记并清理定时器，避免组件卸载后继续更新状态。

这种实现刻意使用 `getContractEvents` 按区块范围轮询，避免某些公共 RPC 节点的 filter 丢失导致 `eth_getFilterChanges` 报错。

### 使用到的 viem 方法

- `publicClient.getBlockNumber()`：获取最新区块号。
- `publicClient.getContractEvents({ address, abi, eventName, fromBlock, toBlock })`：按区块范围读取合约事件。
- `formatUnits(value, 18)`：格式化事件中的 Token 数量。
- `isAddress(value)`：监听前校验合约地址。

## 5. ERC-20 Token 转账

### 实现逻辑

1. 用户输入接收地址和 Token 数量。
2. 按钮启用条件：合约地址、接收地址合法，并且数量大于 0。
3. 点击“转账 Token”后再次校验合约地址，防止绕过按钮状态。
4. 调用 `getWalletAccount()` 获取 `walletClient` 和 `account`。
5. 使用 `parseUnits(amount, 18)` 将 Token 数量转换为最小单位。
6. 调用：
   `walletClient.writeContract({ account, address, abi: erc20Abi, functionName: 'transfer', args: [to, amountInBaseUnit] })`。
7. 展示返回的交易哈希。
8. 调用 `publicClient.waitForTransactionReceipt({ hash })` 等待交易确认。

### 使用到的 viem 方法

- `walletClient.writeContract(...)`：调用 ERC-20 写入函数。
- `parseUnits(amount, 18)`：Token 数量转最小单位。
- `publicClient.waitForTransactionReceipt({ hash })`：等待交易确认。
- `isAddress(value)`：校验合约地址和接收地址。

## 5 个模块与 viem 方法速查表

| 功能模块 | 核心逻辑 | viem 方法 |
| --- | --- | --- |
| 连接测试网并查询 ETH 余额 | 创建钱包客户端获取账号，用只读客户端查余额 | `defineChain`、`custom`、`createWalletClient`、`requestAddresses`、`getBalance`、`formatEther` |
| 发送 ETH | 钱包客户端发原生币交易，只读客户端等回执 | `sendTransaction`、`parseEther`、`waitForTransactionReceipt` |
| ERC-20 `balanceOf` | 按 ABI 调用只读合约函数 | `readContract`、`formatUnits` |
| 监听 `Transfer` | 获取最新区块并按区块范围轮询事件 | `getBlockNumber`、`getContractEvents`、`formatUnits` |
| ERC-20 Token 转账 | 钱包客户端调用 ERC-20 `transfer` | `writeContract`、`parseUnits`、`waitForTransactionReceipt` |
