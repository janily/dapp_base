# ethers 功能实现总结

本文档总结 `ethers` 示例应用中 5 个核心功能模块的实现逻辑，以及每个模块使用到的 ethers v6 方法。应用整体通过 `src/config.ts` 读取链、RPC、ERC-20 合约地址等配置，并在 `src/App.tsx` 中分别实现钱包连接、ETH 查询/转账、ERC-20 查询/监听/转账。

## 公共配置与基础封装

### 配置来源

- `appConfig` 从 `VITE_ETHERS_*` 或通用 `VITE_*` 环境变量读取：`CHAIN_ID`、`CHAIN_NAME`、`RPC_URL`、`READ_RPC_URL`、`ERC20_ADDRESS`、`DEFAULT_QUERY_ADDRESS`、`DEFAULT_RECIPIENT`。
- 默认链为 Sepolia：`chainId = 11155111`，默认公网 RPC 为 `https://ethereum-sepolia-rpc.publicnode.com`。
- `readRpcUrl` 默认使用 `/rpc/sepolia`，方便在部署环境中通过反向代理隐藏或统一转发 RPC 请求。

### 关键 ethers 封装

| 封装/方法 | 作用 | 使用场景 |
| --- | --- | --- |
| `new JsonRpcProvider(rpcUrl, chainId)` | 创建只读 RPC Provider | 查询 ETH 余额、读取 ERC-20、监听事件 |
| `new BrowserProvider(window.ethereum)` | 将浏览器钱包注入的 EIP-1193 Provider 包装为 ethers Provider | 获取 signer、发起签名交易 |
| `provider.getSigner()` | 获取当前钱包账户对应的 signer | ETH 转账、ERC-20 转账 |
| `signer.getAddress()` | 获取 signer 绑定的钱包地址 | 连接钱包后展示当前地址 |
| `new Contract(address, abi, runner)` | 创建合约实例；runner 可以是只读 Provider 或可写 Signer | balanceOf、Transfer 事件监听、transfer |
| `isAddress(value)` | 校验地址是否合法 | 表单按钮禁用、合约地址/账户地址校验 |
| `parseEther(value)` / `formatEther(value)` | ETH 字符串与 wei bigint 互转 | ETH 转账数量、ETH 余额展示 |
| `parseUnits(value, 18)` / `formatUnits(value, 18)` | Token 字符串与最小单位互转 | ERC-20 转账数量、Token 余额展示 |

### 钱包切链逻辑

`switchToConfiguredChain()` 直接调用浏览器钱包的 EIP-1193 RPC：

1. 根据 `appConfig.chainId` 组装十六进制 `chainId`。
2. 调用 `wallet_switchEthereumChain` 尝试切换到配置链。
3. 如果钱包返回 `4902`（未知链），调用 `wallet_addEthereumChain` 添加链配置。
4. 如果没有 `window.ethereum`，抛出“未检测到浏览器钱包”的错误。

### 状态处理

`useStatus()` 是本示例的通用异步状态封装：

1. 执行任务前设置状态文案并清空错误。
2. 任务成功后展示“完成”。
3. 捕获异常后展示错误信息并清空状态。

## 1. 连接测试网并查询 ETH 余额

### 实现逻辑

1. 用户点击“连接钱包”或“切换账号”。
2. 调用 `requestWalletAccount(forceAccountPicker)`：
   - 先执行 `switchToConfiguredChain()`，确保钱包处在配置的测试网。
   - 如果是“切换账号”，通过 `wallet_requestPermissions` 重新请求 `eth_accounts` 权限，让钱包弹出账号选择。
   - 使用 `getBrowserProvider()` 创建 `BrowserProvider`。
   - 通过 `provider.getSigner()` 获取当前账户 signer。
   - 通过 `signer.getAddress()` 读取当前钱包地址。
3. 将钱包地址写入页面状态；如果查询地址为空，则默认填入当前钱包地址。
4. 用户输入要查询的地址后，点击“查询余额”。
5. 使用 `getReadProvider()` 创建只读 `JsonRpcProvider`。
6. 调用 `provider.getBalance(queryAddress)` 读取 ETH 余额。
7. 用 `formatEther(value)` 将 wei 格式的 bigint 转成 ETH 字符串并展示。

### 使用到的 ethers 方法

- `BrowserProvider(window.ethereum)`：连接浏览器钱包。
- `provider.getSigner()`：获取可签名账户。
- `signer.getAddress()`：读取当前钱包地址。
- `JsonRpcProvider(rpcUrl, chainId)`：创建只读 RPC Provider。
- `provider.getBalance(address)`：查询地址的原生币余额。
- `formatEther(value)`：将 wei 格式化为 ETH。
- `isAddress(value)`：校验输入地址是否合法。

## 2. 发送 ETH

### 实现逻辑

1. 用户输入接收地址和 ETH 数量。
2. 按钮启用条件：接收地址通过 `isAddress(to)` 校验，并且数量大于 0。
3. 点击“发送 ETH”后，调用 `requestWalletAccount()`：
   - 确保钱包切换到配置链。
   - 获取当前账户 signer。
4. 使用 `parseEther(amount)` 将 ETH 字符串转换为 wei。
5. 调用 `signer.sendTransaction({ to, value })` 发起原生币转账。
6. 将返回对象中的 `tx.hash` 展示为交易哈希。
7. 调用 `tx.wait()` 等待交易上链确认。

### 使用到的 ethers 方法

- `signer.sendTransaction({ to, value })`：发送原生 ETH 交易。
- `parseEther(amount)`：将 ETH 数量转换成 wei。
- `tx.wait()`：等待交易回执确认。
- `isAddress(value)`：校验收款地址。
- `BrowserProvider` / `getSigner()`：获取可发送交易的 signer。

## 3. 调用 ERC-20 `balanceOf`

### 实现逻辑

1. ERC-20 合约地址来自环境变量或页面中的“配置合约地址”弹窗。
2. 用户输入要查询 Token 余额的账户地址。
3. 按钮启用条件：合约地址和账户地址都通过 `isAddress` 校验。
4. 点击“读取 Token 余额”后：
   - 调用 `getReadErc20Contract(erc20Address)` 创建只读合约实例。
   - 合约实例内部使用 `new Contract(erc20Address, erc20Abi, getReadProvider())`。
5. 调用 `contract.balanceOf(account)` 获取 ERC-20 最小单位余额。
6. 使用 `formatUnits(value, 18)` 按 18 位小数格式化显示。

### 使用到的 ethers 方法

- `new Contract(address, erc20Abi, provider)`：创建只读 ERC-20 合约实例。
- `contract.balanceOf(account)`：调用 ERC-20 只读函数。
- `formatUnits(value, 18)`：把 Token 最小单位格式化为可读数量。
- `isAddress(value)`：校验合约地址和账户地址。
- `JsonRpcProvider`：为只读合约调用提供 RPC 连接。

## 4. 监听 ERC-20 `Transfer` 事件

### 实现逻辑

1. `TransferEvents` 组件在 `erc20Address` 变化时重新初始化监听。
2. 如果合约地址无效，直接返回，不创建监听器。
3. 创建只读 ERC-20 合约实例：`getReadErc20Contract(erc20Address)`。
4. 定义监听函数 `listener(from, to, value)`：
   - 接收 `Transfer` 事件参数 `from`、`to`、`value`。
   - 用 `formatEther(value)` 将事件中的数量格式化展示。
   - 将新事件插入列表顶部，并只保留最近 20 条。
5. 调用 `contract.on('Transfer', listener)` 订阅事件。
6. React effect 清理阶段调用 `contract.off('Transfer', listener)` 取消订阅，避免组件卸载或合约地址变化后重复监听。

> 注意：这里使用 `formatEther(value)` 展示事件数量，等价于按 18 位小数格式化；如果目标 ERC-20 小数位不是 18，应改用对应 decimals。

### 使用到的 ethers 方法

- `new Contract(address, erc20Abi, provider)`：创建带事件 ABI 的合约实例。
- `contract.on('Transfer', listener)`：监听合约事件。
- `contract.off('Transfer', listener)`：取消合约事件监听。
- `formatEther(value)`：把 18 位小数的事件数量格式化为可读字符串。
- `isAddress(value)`：监听前校验合约地址。

## 5. ERC-20 Token 转账

### 实现逻辑

1. 用户输入接收地址和 Token 数量。
2. 按钮启用条件：ERC-20 合约地址、接收地址均合法，并且数量大于 0。
3. 点击“转账 Token”后：
   - 再次校验 ERC-20 合约地址，避免绕过按钮状态。
   - 调用 `requestWalletAccount()` 切链并获取 signer。
   - 使用 `new Contract(erc20Address, erc20Abi, signer)` 创建可写合约实例。
4. 使用 `parseUnits(amount, 18)` 将 Token 数量转换为最小单位。
5. 调用 `contract.transfer(to, amountInBaseUnit)` 发起 ERC-20 转账交易。
6. 展示 `tx.hash`。
7. 调用 `tx.wait()` 等待交易确认。

### 使用到的 ethers 方法

- `new Contract(address, erc20Abi, signer)`：创建可写 ERC-20 合约实例。
- `contract.transfer(to, amount)`：调用 ERC-20 写入函数。
- `parseUnits(amount, 18)`：将 Token 数量转换为最小单位。
- `tx.wait()`：等待交易确认。
- `isAddress(value)`：校验合约地址和接收地址。
- `BrowserProvider` / `getSigner()`：获取签名交易所需 signer。

## 5 个模块与 ethers 方法速查表

| 功能模块 | 核心逻辑 | ethers 方法 |
| --- | --- | --- |
| 连接测试网并查询 ETH 余额 | 切链、获取 signer 地址、用只读 Provider 查询余额 | `BrowserProvider`、`getSigner`、`getAddress`、`JsonRpcProvider`、`getBalance`、`formatEther`、`isAddress` |
| 发送 ETH | signer 发送原生币交易并等待确认 | `sendTransaction`、`parseEther`、`wait`、`isAddress` |
| ERC-20 `balanceOf` | 只读合约调用并格式化 Token 数量 | `Contract`、`balanceOf`、`formatUnits`、`JsonRpcProvider` |
| 监听 `Transfer` | 合约事件订阅，状态保留最近 20 条 | `Contract.on`、`Contract.off`、`formatEther` |
| ERC-20 Token 转账 | signer 绑定合约，调用 `transfer` 并等待确认 | `Contract`、`transfer`、`parseUnits`、`wait` |
